# Claude Project Standards & Instructions

## Overview
This project follows best practices for Python development, focusing on clarity, maintainability, and reliability. All contributors are expected to adhere to the standards outlined below.

---

## Project Structure
- Organize code into logical modules and packages.
- Use descriptive folder and file names.
- Separate notebooks, scripts, and utility files.
- Keep documentation and media assets in dedicated folders.

---

## Coding Standards
- **PEP 8 Compliance:** Follow [PEP 8](https://peps.python.org/pep-0008/) for code style.
- **Type Annotations:** Use type hints for functions and classes.
- **Docstrings:** Document all public functions, classes, and modules using [PEP 257](https://peps.python.org/pep-0257/).
- **Naming:** Use clear, descriptive names for variables, functions, and classes.
- **Functions Naming:** Use `snake_case` for functions and methods.
- **Error Handling:** Use exceptions thoughtfully; catch only what you can handle.
- **Immutability:** Prefer immutable data structures when possible.

---

## Dependency Management
- Use `requirements.txt` and/or `pyproject.toml` to specify dependencies.
- Pin versions for critical packages.
- Use virtual environments (e.g., `.venv`) for isolation.

---

## Documentation
- Maintain clear and concise README files.
- Use Markdown for all documentation.
- Document setup, usage, and contribution guidelines.

---

## Version Control
- Commit often with meaningful messages.
- Use branches for new features and bug fixes.
- Keep the main branch stable.

---

## Notebooks
- Keep notebooks clean and well-commented.
- Use Markdown cells for explanations.
- Avoid storing large outputs or data in notebooks.

---

## Contribution Guidelines
- Follow the standards above.
- Review code before submitting pull requests.
- Be respectful and collaborative.
